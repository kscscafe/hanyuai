# /session_end — セッションまとめ自動生成

このコマンドが呼ばれたら、以下を順番に実行する。

---

## Step 1: 今日の作業を振り返る

この会話の中で行った作業を以下の観点で整理する：
- 実装・修正したファイル
- 決定した設計・仕様
- 発見したバグ・課題
- 次回に持ち越すこと

---

## Step 2: まとめファイルを生成する

以下のフォーマットで `docs/sessions/YYYYMMDD.md` を生成する。
同日2件目は `YYYYMMDD-b.md`、3件目は `YYYYMMDD-c.md` とする。

```markdown
# セッションまとめ
_YYYY年M月D日_

## 1. このセッションでやったこと（一言）

## 2. 作業内容

### [作業タイトル]
**変更ファイル**
**内容**

## 3. 決定事項・ナレッジ

## 4. 発見した課題

## 5. 次回やること
```

---

## Step 3: TASKS.md を更新する

- 今日完了したタスクに `[x]` をつける
- 新たに発見した課題を追記する

---

## Step 4: コミットする

```
git add docs/sessions/YYYYMMDD.md TASKS.md
git commit -m "docs: セッションまとめ YYYYMMDD"
git push
```

---

## 注意事項
- まとめは事実ベースで書く
- 理由・判断の根拠も残す
- 個人情報・APIキー・パスワードは絶対に書かない
